# Team Onboarding — iOS SDK

Guide for engineers joining the BidsCube iOS SDK team.

## Prerequisites

- macOS with **Xcode 15+**
- **CocoaPods** 1.10+ (`gem install cocoapods`)
- Access to this Git repository
- Optional: CocoaPods Trunk credentials (release owners only)

## First-time setup

```bash
cd bidscube-sdk-ios
pod install
open bidscubeSdk.xcworkspace    # NOT .xcodeproj alone when using Pods
```

Always build from **`bidscubeSdk.xcworkspace`** — it links the test app with Pods (AppLovin, IMA via Pods path) and the SDK framework.

## Xcode schemes

| Scheme | Product | Use for |
|--------|---------|---------|
| **bidscubeSdk** | `bidscubeSdk.framework` | SDK development, Release build verification |
| **testApp-ios** | QA app | Manual testing, video interstitial QA tab |

Command-line build:

```bash
xcodebuild -workspace bidscubeSdk.xcworkspace \
  -scheme bidscubeSdk \
  -destination 'generic/platform=iOS Simulator' \
  build
```

## Critical architectural rule

**Two separate video implementations exist:**

| Path | Entry API | Implementation |
|------|-----------|----------------|
| **Interstitial video (new)** | `showInterstitialVideoAd` | `VideoInterstitial/*` |
| **Rewarded video (legacy)** | `showRewardedVideoAd` | `AdViewController` + `IMAVideoAdHandler` |

Do not mix them. Rewarded needs `onUserRewarded` on IMA `.COMPLETE` only — the interstitial module does not implement rewards.

## Making changes safely

1. Match existing naming and patterns in the file you edit.
2. If changing callback timing, update [callbacks-and-events.md](callbacks-and-events.md) and notify QA.
3. If changing VAST behaviour, update `VastMetadataParserTests` and [video-interstitial-qa.md](../qa/video-interstitial-qa.md).
4. Never commit `build/`, `.build/`, or `Pods/` unless intentionally updating dependencies.
5. Version bumps: see [build-release-ci.md](build-release-ci.md).
